## 自己 PR

**1. 設計から運用定着まで、一人称で推進する実行力**
コードキャンプでは、社内のインフラ／プラットフォーム領域の主担当として、構想から実装・運用定着まで一貫して推進してきました。担当領域は以下のように多岐に渡ります：

- AWS WAF / Security Hub の導入と運用整備
- MySQL → BigQuery の ETL 基盤構築（Embulk + digdag）
- CloudFormation + Ansible によるインフラ管理の標準化
- EC2 → ECS Fargate 移行、CircleCI → GitHub Actions 移行
- RSpec System Spec / Capybara による E2Eテスト導入

技術選定から優先順位付け、関係者調整まで自ら設計・判断し、限られたリソースの中で確実に成果を出してきました。

**2. 信頼性と開発体験を両立させるプラットフォームエンジニアリング視点**
プラットフォームの信頼性向上だけでなく、CI管理の一元化・保守性向上やテスト運用の整備など、「開発者が速く安全にリリースできる環境」を作ることを一貫して重視してきました。本番障害発生時はインシデント対応の主担当として原因特定・暫定対処・恒久対応までをリードしており、運用品質と開発体験の双方に直結するプラットフォーム改善を設計することを意識しています。

**3. 生成 AI 活用の実践経験**
生成AI全社活用プロジェクトのPLとして、ツール選定から社内展開まで横断的に担当。n8n・Difyを用いたワークフロー自動化を3本実装し、本番運用に乗せました。インフラ運用や開発フローの自動化・効率化にも、AI活用の視点を取り入れられます。

**4. 技術発信と知識のアウトプット経験**
Qiitaでの技術記事発信、法人研修向けクラウド・AWS教材の執筆およびLinux教材のレビュー実績があります。自分の知見を言語化して他者に伝える力は、社内ドキュメンテーションや勉強会運営にも直接活かせると考えています。

**5. 今後の志向**
インフラ／プラットフォーム領域の実装力を軸に、引き続き会社・組織・プロダクトの価値が最大化される方向で動いていきたいです。技術の選択は手段なので、事業や開発チームにとって何が本当に効くかを考えながら、アウトプットし続けたいと思ってます。
